# Product Invariants

Replace this template with durable product principles that should survive implementation changes.

Keep this file small. Do not record every feature or current file path here.

## Invariants

- The product should solve the user's real workflow, not only expose technical capability.
- User-facing copy should sound natural and product-specific.
- Data safety, privacy, and user trust take priority over implementation convenience.
- Current implementation shape may change when the task calls for it.

## Add Project-Specific Items

Examples:

- Offline-first or local-first constraints.
- Privacy promises.
- Naming and brand boundaries.
- Subscription or pricing principles.
- Critical accessibility or reliability expectations.
