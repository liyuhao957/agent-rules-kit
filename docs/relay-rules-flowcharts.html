Hard block 1: bash guard

pre_bash_release_guard.py fails closed: if a risky command or unparsable command appears, it exits 2 unless the risk bypass is deliberately set.

Hard block 2: Stop gate

stop_quality_reminder.py refreshes candidates at finalization and blocks only unresolved high-risk items: secrets, billing, release, remote or production state.

Boundary: validation checks form, not factual correctness. Generated from repo-local evidence, not old notes.

收尾门禁安全结束

只拦未处理的高风险 risk:* 候选。普通 drift/command 候选只提示，不硬拦。

RULES_HOOK_ALLOW_PENDING=1
会话结束后，下次从第 6 步重新开始。

硬拦截 1：bash 守卫

pre_bash_release_guard.py 失败时宁可拦住：命令高风险或解析不了，都会 exit 2，除非明确设置风险旁路。

硬拦截 2：收尾门禁

stop_quality_reminder.py 收尾时刷新候选，只拦密钥、计费、发布、远程或生产状态这类未处理高风险事项。

边界：校验只查形式，不证明事实本身正确。 来源是当前仓库证据，不是旧说明或旧记忆。
